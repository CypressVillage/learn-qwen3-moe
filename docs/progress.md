形状推理提示：

本模板统一使用 `S` 表示本次 forward 的输入/query 长度，`P` 表示调用前的缓存历史长度，`T=P+S` 表示 attention 使用的 key/value 总长度。无 KV Cache 时 `P=0`、`T=S`。

- 哪些维度被保留、拆分、合并或收缩？
- 是否发生广播？广播前两个操作数的 shape 分别是什么？
- `reshape`/`transpose` 后 stride 和 contiguous 状态是否重要？
- Attention 中 `Hq` 与 `Hkv` 是否不同？为什么 query 是 `[B,Hq,S,Dh]`、attention 使用的 key/value 是 `[B,Hkv,T,Dh]`、scores 是 `[B,Hq,S,T]`？
- MoE 中何时把 `[B,S,D]` 展平为 `[N,D]`？`N=B*S` 是否始终成立？
- Top-K 的 indices 和 weights 为什么是 `[N,K]`？
- KV Cache 在追加前是否为 `[B,Hkv,P,Dh]`，追加当前 K/V 后是否为 `[B,Hkv,T,Dh]`？是否误把历史长度 `P`、当前长度 `S` 和总长度 `T` 混用？
